TradingBot

Multi-stock ML trading bot

All S&P 500 stocks pass through a single decoder-only Transformer in one forward pass. Learns inter-stock relationships — outputs per-stock buy/sell confidence scores from -1 to +1.

View on GitHub Quick Start

Features

Encoder-Only Transformer

Full bidirectional attention — every stock attends to every other stock. Research shows this architecture captures cross-stock relationships effectively. ~478K parameters.

Multi-Window Features

30 technical indicators (SMA, RSI, MACD, Bollinger, volatility, returns, drawdown) at 1y, 1m, 1w, 1d lookbacks — 120 features per stock per day. Crypto pipeline uses BTC/USD as market proxy.

Portfolio-Aware Training

From MSE to MSRR (direct Sharpe optimization), margin ranking, and ListNet losses. MSRR optimizes portfolio weights, not individual stock errors.

Market-Guided Gating

SPY state (returns, volatility, volume) rescales features before the transformer. The model adapts to bull/bear/high-volatility regimes automatically.

How It Works

  • 1. Data — Downloads OHLCV for all S&P 500 stocks via yfinance (~10 years), cached to CSV. Tickers fetched live from Wikipedia.
  • 2. Features — Builds parallel feature matrix: 30 technical indicators per stock per date at 4 lookback windows (1y, 1m, 1w, 1d). SMA, RSI, MACD, Bollinger Bands, volatility, returns, max drawdown, volume ratio.
  • 3. Model — Encoder-only Transformer with full bidirectional attention. Every stock attends to every other stock. MarketGate rescales features by SPY state. RankGLU output head produces confidence scores.
  • 4. Training — Supervised regression on next-day cross-sectional z-score returns. Supports MSE, MSRR (direct Sharpe optimization), margin ranking, and ListNet losses. Mixed precision, gradient accumulation, cosine annealing LR, early stopping.
  • 5. Threshold — Post-training calibration via dual-threshold search maximizes Sharpe ratio. Separate buy and sell thresholds. Paper trading loop runs via trade.py (Rich) or textual_trader.py (TUI).

Terminology

  • MSE — Mean Squared Error per stock. Simple regression baseline: loss = Σ(predicted - actual)².
  • MSRR — Max Sharpe Ratio Regression. Loss = (1 - portfolio_return)². Directly optimizes portfolio Sharpe. Noisier but higher ceiling.
  • Margin / ListNet — Pairwise ranking loss (margin: correct relative order) and listwise ranking loss (ListNet: top-1 probability distribution). Good for ranking tasks.
  • Mixed Precision — float16 for compute-heavy ops, float32 for critical ops. 30-40% speedup on CUDA and MPS with no accuracy loss.
  • RankGLU — Residual bottleneck GLU output head: direct linear path + gated nonlinear branch. Preserves ranking while adding controlled interactions.
  • MarketGate — SPY market features (returns, volatility, volume ratio) rescale stock features before the transformer. Regime-adaptive conditioning from MASTER (AAAI 2024).
  • Walk-Forward — Multiple chronological train/val/test folds evaluated sequentially. Gold standard for financial ML — captures regime changes.
  • Z-Score Norm — Training targets normalized per-day: mean=0, std=1 after winsorizing extremes. Standard in ranking-aware models.

Quick Start

# 1. Install dependencies
uv sync

# 2. Train the model (MSE baseline)
uv run python main.py --mode train

# 3. Train with MSRR loss (gradient accumulation)
uv run python main.py --mode train --loss msrr --grad-accum 4

# 4. Ensemble training (5 seeds)
uv run python main.py --mode train --loss msrr --seeds 5 --grad-accum 4

# 5. Walk-forward validation
uv run python main.py --mode train --walk-forward

# 6. Get today's trading signals
uv run python main.py --mode infer

# 7. Paper trading (Rich dashboard)
uv run python trade.py --interval 15

# 8. Paper trading (Textual TUI)
uv run python textual_trader.py --interval 15

# 9. Crypto mode
uv run python trade.py --interval 15 --asset-class crypto

# 10. Multi-GPU training
torchrun --nproc_per_node=N uv run python main.py --mode train
